\# i18n — Internationalisation Design

**Date:** 2026-04-17
**Status:** Approved

## Goal

Extract all hardcoded user-visible strings from the module and move them into FoundryVTT translation JSON files. Launch with English and French. Keep LLM prompt content (system/user prompts sent to AI providers) hardcoded — those are instructions to the model, not UI strings.

## Approach

Flat `lang/en.json` / `lang/fr.json` files under the `COC7QOL.` key prefix. A thin `t()` / `tf()` helper exported from `utils.js` wraps `game.i18n.localize()` and `game.i18n.format()` to keep call sites concise.

## Infrastructure

### New files

- `lang/en.json` — English strings
- `lang/fr.json` — French strings

### `module.json` addition

```json
"languages": [
  { "lang": "en", "name": "English",   "path": "lang/en.json" },
  { "lang": "fr", "name": "Français",  "path": "lang/fr.json" }
]
```

### `scripts/utils.js` additions

```js
export const t  = key         => game.i18n.localize(key)
export const tf = (key, data) => game.i18n.format(key, data)
```

## Code changes per file

### `scripts/utils.js`
Add `t()` and `tf()` exports at the bottom of the file.

### `scripts/close-all-cards.js`
- Import `t`, `tf` from `./utils.js`
- `CARD_TYPE_LABELS` map: replace string values with `t()` calls. **Important:** these must be resolved at use-time (inside methods), not at module load time, because `game.i18n` is not ready during static initialisation.
- `actor: message.speaker?.alias ?? 'Unknown'` → `?? t('COC7QOL.CloseAllCards.UnknownActor')`
- Window `title` in `DEFAULT_OPTIONS` and `title` getter → `t('COC7QOL.CloseAllCards.Title')`
- Header count: `tf(count === 1 ? 'COC7QOL.CloseAllCards.Header.OpenCard' : 'COC7QOL.CloseAllCards.Header.OpenCards', { count })`
- Button labels in `footer.innerHTML`: `t()` inline in the template literal
- `ui.notifications.info(...)` calls → `tf()` with count, or `t()` for no-cards case

### `scripts/ai-generator/dialog-injector.js`
- Import `t`, `tf` from `../utils.js`
- `WEAPON_CONFIG` and `NPC_CONFIG` object literals: replace `label`, `placeholder`, `prefillPrefix` string values with `t()` calls. Since these are top-level `const` objects evaluated at module load time, convert each translated property to a getter (e.g. `get label() { return t('...') }`) so resolution is deferred until the hook fires.
- `aiBtn.title` assignments → `t()`
- `buttonRow.innerHTML` button labels → `t()` in template literal
- `generateBtn.textContent` assignments → `t()`
- `errorDiv.textContent` assignments → `t()` / `tf()`
- `ui.notifications.*` calls → `t()` / `tf()`

### `scripts/ai-generator/generation-dialog.js`
- Import `t` from `../utils.js`
- Remove `window: { title: '...' }` from `DEFAULT_OPTIONS`; add `get title() { return t('COC7QOL.AIGenerator.WeaponDialog.Title') }` getter on the class
- Replace `<label>`, `<dt>` text, and button labels in `div.innerHTML` template with `t()` calls

### `scripts/ai-generator/npc-confirmation-dialog.js`
- Import `t` from `../utils.js`
- Remove `window: { title: '...' }` from `DEFAULT_OPTIONS`; add `get title()` getter
- Characteristic labels array: replace `['STR', 'CON', ...]` with `['CHARAC.STR', 'CHARAC.CON', ...]` resolved via `t()` at render time
- `Occupation` / `Age` labels → `t('CoC7.Occupation')` / `t('CoC7.Age')`
- Section labels (`Characteristics`, `Skills`, etc.) → `t()`
- Button labels in `buttonsHtml` → `t()`

### `scripts/ai-generator/settings.js`
- Replace `name:` and `hint:` string literals with their key strings (e.g. `name: 'COC7QOL.Settings.AIProvider.Name'`). FoundryVTT calls `game.i18n.localize()` on these automatically — no `t()` needed here.
- Replace provider `choices` values with their key strings.

## Timing constraint

`game.i18n` is not available at module load time (static class field initialisers, top-level `const` declarations). All `t()` / `tf()` calls must happen inside methods or event handlers that run after the `init` hook. The two dialog titles are the main risk — solved by replacing `DEFAULT_OPTIONS.window.title` with a `get title()` getter, which is called at render time.

## Skill names (NPC dialog)

Skill names displayed in the NPC confirmation dialog are dynamic data resolved from the `CoC7.skills` compendium by the NPC mapper. Compendium item names are already localised by the CoC7 system. No i18n keys needed for skill names.

## Card type labels (Close All Cards)

The `CARD_TYPE_LABELS` map keys are CoC7 internal template class names (e.g. `CoC7SanCheckCard`). These map to short display labels that we own and translate ourselves. The CoC7 system has some overlapping keys but with different wording (e.g. "Sanity check" vs our "Sanity Check"); we keep our own for consistency.

## Out of scope

- LLM system/user prompt text (AI instructions, not UI)
- Console error/warning messages (`console.error`, `console.warn`)
- Languages beyond English and French at launch
